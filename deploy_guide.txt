====================================================================
			  GUIDE DE DEPLOIEMENT - HEALTHAI COACH (MSPR)
====================================================================

Ce document explique comment etre operationnel rapidement sur les 3
briques du projet:

- API Laravel + PostgreSQL
- ETL Python (ingestion de donnees)
- Grafana (visualisation)
- API IA FastAPI (Analyse d'images via LLaVA)

Le mode d'emploi est optimise pour Windows + Docker Desktop.

--------------------------------------------------------------------
1. VUE D'ENSEMBLE
--------------------------------------------------------------------

Le projet est compose de 3 stacks Docker:

1. HealthAI-Coach
	- Laravel (API)
	- PostgreSQL

2. ETL
	- Pipeline Python
	- Grafana

3. API-IA
	- FastAPI
	- Ollama (LLaVA) - A FAIRE

Point d'entree recommande:

- Lancer start.bat a la racine du workspace.

Ce script automatise:

1. verification de l'environnement (Docker, daemon, ports)
2. demarrage API + PostgreSQL avec healthchecks
3. attente du demarrage de PostgreSQL ET de Laravel
4. reparation PostgreSQL automatique si ancien cluster detecte
5. migrations + seeders Laravel
6. demarrage ETL + Grafana
7. demarrage API IA FastAPI (optionnel, si Ollama est orchestré ici)

--------------------------------------------------------------------
2. PREREQUIS
--------------------------------------------------------------------

Pour executer le projet:

- Windows
- Docker Desktop en cours d'execution
- Docker Compose v2 disponible (commande: docker compose)

Ports utilises:

- API: 80
- PostgreSQL expose Windows: 55432
- Grafana: 3000
- API IA FastAPI: 4000

Important:
Le port 55432 est volontaire pour eviter les conflits avec un PostgreSQL
local Windows sur 5432.

--------------------------------------------------------------------
3. DEMARRAGE RAPIDE (RECOMMANDE)
--------------------------------------------------------------------

1. Ouvrir le dossier racine du projet.
2. Initialiser les fichiers d'environnement si absents:
	- API Laravel: copier HealthAI-Coach/.env.example vers HealthAI-Coach/.env
	- ETL: copier ETL/.env.example vers ETL/.env
3. Double-cliquer sur start.bat.
4. Attendre le message de fin.

3.1 Usages de start.bat

Le script accepte des arguments optionnels combinables:

- start.bat
	Demarrage standard interactif. Le script attend une touche a la fin.

- start.bat --auto
	Mode CI/CD non-interactif. Pas de pause a la fin du script.
	Utile pour les pipelines automatises.

- start.bat --fresh
	Reset complet de la base de donnees avant demarrage.
	Execute migrate:fresh --force --seed : supprime toutes les tables,
	relance les migrations et le seeding depuis zero.
	ATTENTION : toutes les donnees existantes sont perdues.

- start.bat --auto --fresh
	Combine les deux modes : reset complet de la BDD en mode
	non-interactif. Ideal pour un environnement de test CI/CD propre.

Controles effectues au demarrage:

1. Verification que Docker est installe (docker --version)
2. Verification que docker compose est disponible
3. Verification que le daemon Docker repond (docker info)
4. Attente des healthchecks des conteneurs (--wait)
5. Attente que PostgreSQL accepte les connexions (pg_isready)
6. Attente que le conteneur Laravel repond (php -r "echo 'ok'")

Liens de verification:

- API: http://localhost
- BackOffice: http://localhost/admin
- Swagger: http://localhost/api/documentation
- Grafana: http://localhost:3000
- API IA (FastAPI): http://localhost:4000/docs
Identifiants Grafana par defaut:

- user: admin
- password: admin

6. ETL - CONFIGURATION OBLIGATOIRE
--------------------------------------------------------------------

L'ETL recupere des fichiers depuis Google Drive puis charge les donnees
dans PostgreSQL.

Fichiers utiles:

- ETL/.env
- ETL/.env.example
- ETL/main.py
- ETL/config.py
- ETL/utils/driveHelper.py
- ETL/README.md

6.1 Variables ETL attendues (.env)

- DATABASE_URL
- GCS_BUCKET
- TO_IMPORT_ID
- ARCHIVE_ID
- ERROR_ID
- LOG_ID
- GOOGLE_TOKEN_PICKLE

Exemple DATABASE_URL (depuis le conteneur ETL):

- postgresql://sail:password@host.docker.internal:55432/laravel

6.2 Arborescence Google Drive obligatoire

Creer cette structure cote Google Drive:

- ETL/
- ETL/ToImport/
- ETL/Archive/
- ETL/Error/
- ETL/Log/

Recuperer les IDs des 4 sous-dossiers et les renseigner dans ETL/.env:

- TO_IMPORT_ID
- ARCHIVE_ID
- ERROR_ID
- LOG_ID

6.3 OAuth Google Drive (credentials.json)

1. Aller sur https://console.cloud.google.com/apis/credentials.
2. Creer un OAuth 2.0 Client ID.
3. Generer puis telecharger le fichier JSON du client OAuth.
4. Renommer le fichier en credentials.json.
5. Placer credentials.json a la racine du dossier ETL.

6.4 Initialisation du token Drive (premiere execution)

1. Ouvrir ETL/utils/driveHelper.py.
2. Decommenter temporairement le code d'initialisation dans la fonction get_drive_service.
3. Lancer l'ETL: une fenetre Google s'ouvre pour autoriser le compte.
4. Verifier que token.pickle est genere.
5. Optionnel: recommenter la partie d'initialisation apres creation du token.

6.5 Option token en Base64

Vous pouvez encoder token.pickle en Base64 de deux manieres:

- Via un script Python (exemple):
	python -c "import base64;print(base64.b64encode(open('token.pickle','rb').read()).decode())"

- Via une commande Linux (exemple):
	base64 -w 0 token.pickle

Copier ensuite la chaine Base64 generee et la coller dans la variable
GOOGLE_TOKEN_PICKLE du fichier ETL/.env.

6.6 Logique de traitement des fichiers

- Deposer les fichiers a traiter dans ETL/ToImport/.
- En cas de succes, les fichiers sont deplaces dans ETL/Archive/ avec un
	nom enrichi (date/heure).
- En cas d'erreur, les fichiers sont deplaces dans ETL/Error/.
- Les details d'execution et d'erreur sont traces dans ETL/Log/.

Preparation resumee ETL:

1. Copier ETL/.env.example vers ETL/.env puis renseigner les variables.
2. Creer l'arborescence ETL/ToImport, Archive, Error, Log dans Drive.
3. Ajouter credentials.json dans le dossier ETL.
4. Initialiser le token OAuth puis lancer le pipeline.

10. TROUBLESHOOTING
--------------------------------------------------------------------

A) Je n'arrive pas a me connecter en localhost

Verifier:

1. que vous utilisez localhost:55432 (pas 5432)
2. que la datasource IDE est bien a jour
3. que Docker Desktop est demarre

B) Ancien cluster PostgreSQL incompatible

Le script start.bat tente une reparation auto via:

- HealthAI-Coach/docker/repair-postgres.sql

Si echec, reset propre:

	cd HealthAI-Coach
	docker compose down -v
	docker compose up -d --force-recreate
	docker compose exec -T laravel.test php artisan migrate --seed

C) L'ETL ne traite pas les fichiers

Verifier:

1. IDs Google Drive dans ETL/.env
2. presence de credentials.json dans ETL/
3. token OAuth Google (token.pickle ou GOOGLE_TOKEN_PICKLE)
4. DATABASE_URL
5. que les fichiers sont bien dans ToImport (et non a la racine)
6. logs ETL dans le dossier Drive Log

D) Grafana affiche "connection refused" sur localhost:55432

Cause:

- Grafana tourne en conteneur, localhost pointe vers le conteneur lui-meme.

Correctif:

1. Ouvrir la datasource PostgreSQL dans Grafana
2. Remplacer Host URL par host.docker.internal:55432
3. Sauvegarder avec Save and Test
